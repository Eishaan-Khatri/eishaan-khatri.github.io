# Claim Risk Ledger

This file prevents resume and portfolio inflation. Recruiters forgive narrow scope. They do not forgive claims that collapse under one follow-up question.

## Safe Claims

Use these confidently.

- Published/accepted ESWA Hinglish offensive language detection work.
- 6,800+ Hinglish dataset for low-resource/code-mixed offensive language analysis.
- Span/target-level annotation and structured target-argument-harmfulness framing.
- Evaluation with multilingual transformers such as mBERT, XLM-R, and MuRIL.
- CLIP ViT-B/32 zero-shot robustness evaluation on CIFAR-10-C, CIFAR-100-C, and EuroSAT.
- TTA study on CIFAR-10-C with Source-only, BN Adapt, Tent, UG-Tent, and source-mix variants.
- Source Mix Tent 0.75 improving mean accuracy to 82.64% and reducing negative-transfer cases from 5 to 3 in that study.
- Edge-VCA as a computer-vision analytics prototype using YOLOv8-Nano and OpenCV.
- Error analysis, ablations, computational profiling, and reproducible documentation as part of research practice.

## Use With Caution

Use only with careful wording.

- **CASML 68% hallucination reduction:** keep if the paper/poster supports it. Say "reported" or "in our evaluation" rather than presenting it as universal.
- **Edge-VCA FPS/memory numbers:** include only with "in project benchmark" wording.
- **Industrial anomaly detection:** strong project, but link proof before making it a headline.
- **Polka Labs agents:** strong applied systems signal, but avoid vague "autonomous at scale" phrasing unless the exact deployment is documented.
- **Data science forecasting:** use as feature engineering/modeling proof, not as predictive mastery.

## Do Not Use Without New Evidence

- "State of the art" or "SOTA" unless directly benchmarked against current accepted baselines.
- "Clinical-grade" unless there is documented clinical validation and compliance review.
- "Production-grade" unless the system was deployed, monitored, and used by real users.
- "0 false positives," "0 ID switches," or perfect metrics as headline claims.
- Stock forecasting headline claim `R2 = 0.9998`; this looks like leakage or evaluation fragility unless heavily explained.
- Unsupported BCI/motor-imagery claims unless local source files/results are verified.
- Diffusion/generative-model expertise unless backed by a real implementation or project.
- "Full spectrum of AI"; it makes the profile look unfocused.
- Simulated GitHub contribution heatmap unless explicitly labeled illustrative.

## Safer Rewrites

| Risky Wording | Safer Wording |
|---|---|
| Full spectrum of AI | Work clusters around multilingual NLP, robustness evaluation, and applied AI systems |
| Top-tier venues | Peer-reviewed/selected research venues |
| Novel framework | Span-level annotation and structured extraction framework |
| Production chatbot | Applied chatbot/agent system, with deployment details only if documented |
| Clinical-grade assistant | Protocol-aware mental-health MVP/prototype |
| Stock model achieved R2 0.9998 | Time-series feature engineering and forecasting study with leakage-aware validation needed before external claims |
| SOTA method | Reproducible study or baseline extension |
