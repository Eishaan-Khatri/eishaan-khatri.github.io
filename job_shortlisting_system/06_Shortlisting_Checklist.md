# Shortlisting Checklist

Use this before submitting any application.

## Resume

- [ ] Top half page clearly matches one role.
- [ ] Summary line names the target role family.
- [ ] Skills are ordered for the JD, not alphabetically.
- [ ] Only 2-3 major projects are prominent.
- [ ] Metrics are evidence-backed.
- [ ] No unsupported SOTA, clinical, production, or perfect-score claims.
- [ ] Resume is one page for industry roles.
- [ ] Research CV is one to two pages only when appropriate.

## Portfolio

- [ ] Resume button leads to role-specific resume options.
- [ ] Public identity is coherent.
- [ ] Best evidence projects are linked.
- [ ] Simulated visuals are labeled honestly.
- [ ] Stock forecasting and similar fragile claims are not headline proof.

## LinkedIn

- [ ] About section matches portfolio identity.
- [ ] Featured links show strongest proof.
- [ ] Top skills are role-relevant.
- [ ] Old generic org/member content does not dominate.

## GitHub

- [ ] Featured repos have useful README files.
- [ ] Result CSVs/figures are committed for research projects.
- [ ] Claims in README match claims in resume.
- [ ] Empty or weak repos are not featured.

## Interview Readiness

- [ ] Every bullet can be explained in 60 seconds.
- [ ] Every metric has a limitation attached.
- [ ] You can explain what you personally did.
- [ ] You can explain what you did not do.
