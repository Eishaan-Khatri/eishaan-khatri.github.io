# Job Shortlisting System

This folder turns the portfolio into a controlled application system. The goal is to stop presenting every project at once and instead choose the strongest evidence for the specific role being targeted.

## What This System Solves

- Too many CV variants without a clear usage rule.
- Strong evidence scattered across PDFs, READMEs, CSVs, and application notes.
- Portfolio wording that is broader than what recruiters can understand quickly.
- Risky claims that may be hard to defend in interviews.
- Resume, LinkedIn, GitHub, and portfolio drift.

## Use This Order

1. Start with `00_Master_Evidence_Bank.md`.
2. Check claims against `01_Claim_Risk_Ledger.md`.
3. Pick the role bucket in `02_Role_Target_Resume_Matrix.md`.
4. Apply the workflow in `03_Job_Application_Workflow.md`.
5. Update public-facing copy using `04_Portfolio_LinkedIn_Upgrade.md`.
6. Use `05_Shortlisting_Graph.md` or `05_Shortlisting_Graph.mmd` as the visual map of the whole profile.

## Default Positioning

Use this as the stable public identity:

> AI/ML researcher and engineer working on multilingual NLP, robustness evaluation, and applied AI systems.

This is broad enough to cover the real work, but narrow enough to avoid looking like a random project collector.

## Non-Negotiable Rule

For job shortlisting, do not send a generic resume. Every application must choose one role bucket and make the first half page match that bucket.
