# Job Application Workflow

Use this before every application. It is designed for shortlisting, not ego coverage.

## Step 1: Classify The Job

Pick one bucket:

- NLP / LLM
- Computer Vision / Robust ML
- AI Systems / Agents
- Data Science / Analytics
- Behavioral AI / Product Research
- Research Assistant / Lab

If the job looks like two buckets, choose the one that appears most in the job description title and first five requirements.

## Step 2: Extract Keywords

Copy 10-15 keywords from the JD into a scratch note.

Prioritize:

- frameworks
- model families
- tasks
- datasets
- evaluation metrics
- deployment constraints
- domain words

Do not keyword-stuff. Use only keywords you can defend.

## Step 3: Pick The Resume Variant

Use `02_Role_Target_Resume_Matrix.md`.

If no existing PDF fits, create a new variant from the closest one. Do not send a generic broad CV.

## Step 4: Patch Only Four Areas

Change only:

1. Summary line.
2. Skills order.
3. Project order.
4. Two to four bullets.

Do not rewrite the entire resume every time. That creates drift.

## Step 5: Run The Shortlisting Tests

### Six-Second Test

Can a recruiter tell the role fit from the top half page?

If no, the resume is too broad.

### ATS Test

Do the exact JD terms appear naturally?

If no, add real matching terms to skills or bullets.

### Evidence Test

Can every metric be traced to a source?

If no, remove the number or soften the claim.

### Consistency Test

Does the portfolio tell the same story as the resume?

If no, update the portfolio link or use a different resume.

### Interview Test

Can you explain each bullet in 60 seconds using:

- problem
- method
- result
- limitation
- your contribution

If no, rewrite the bullet.

## Step 6: Application Note Template

Use this short message when a form or email asks for extra information:

> My strongest fit for this role is the combination of research execution and applied ML engineering. I have worked on multilingual NLP and harmful-content detection through my Hinglish ESWA work, and I have also built reproducible robustness studies around test-time adaptation and CLIP under corruption/domain shift. I am strongest when the role needs someone who can read papers, implement baselines, run careful evaluations, and turn results into clear technical artifacts.

## Step 7: Weekly Maintenance

Every Sunday:

- Archive duplicate/outdated resume PDFs.
- Add any new proof to `00_Master_Evidence_Bank.md`.
- Move risky claims to `01_Claim_Risk_Ledger.md`.
- Update the portfolio only if the claim is stable.
